 
HEAD - файлы в рабочей дериктории выглядат так как те файлы на которые указывает HEAD
checkout - перекидывает указатель HEAD

1. Временно переключиться на другой коммит
	git checkout 1077a72
	
2. Переключиться на коммит и продолжить работу с него	
	git checkout -b new-branch-name 1077a72
	
3. Перейти к более раннему коммиту в текущей ветке
	git reset --soft 1077a72
	git reset [--mixed] 1077a72
	git reset --hard 1077a72
	
	
5.  git revert отменяет уже опубликованные коммиты с помощью новых коммитов.
	Создает новые коммиты, по одному на каждый отменяемый коммит. 
	Таким образом, если нужно отменить все коммиты после:
	Допустим aaaaaa - это ID коммита
	
	#Пример git revert HEAD~2 
	
	# можно перечислить отменяемые коммиты
	git revert bbbbbb cccccc dddddd

	# можно задать диапазон от более раннего к более позднему (новому)
	git revert bbbbbb..dddddd

	# либо в относительных ссылках
	git revert HEAD~2..HEAD
