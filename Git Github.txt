1. Создаем новй репозиторий на github.com 

2. Добавит имя для удлаенного репозитория в список удаленных репозиториев
	#git remote add origin https://github.com/mzhelyaskov/new-repo.git
	послей этой комманды вместе адреса репозитория можно будет просто указывать его имя origin.
	
	Для просмотра списока доступных удаленных репоизиториев можно набрать команду
	#git remote 
	
3. Для того чтобы "вытолкнуть" все изменения в удаленный репозиторий необходимо выполнить комманду
	#git push -u origin master  
		-u		говорит гиту запомнить параметры, поэтму в слудующий раз можно выполняь просто git push
		origin	название (адрес) удаленного репозитория
		master  ветка удаленного репозитория на которую "вытолкнутся" изменения.
		
4. Для "втягивания" последних изменений с удаленного репозитория применяется команда pull
	#git pull origin new-branch
		origin		название (адрес) удаленного репозитория
		new-branch	ветка в удаленном репозитории на которую будут вытолкнуты изменения.
					(если в удаленном репоизтории такой ветк нет - то она будет создана автоматически.)
					
5. Актуализация веток на основании данных из других веток с помощью rebase.
	Находясь например на ветке new-branch можно выполнить команду
	#git rebase master	берет все актуальные данные с master и пытается применить их на ветку new-branch
